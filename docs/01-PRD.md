# MeetingMod - Product Requirements Document (PRD)

## 1. 개요

### 1.1 제품명
**MeetingMod** - AI Meeting Moderator Agent

### 1.2 한 줄 요약
회의 전 준비 점검부터 회의 중 실시간 개입, 회의 후 피드백까지 End-to-End로 회의 품질을 높여주는 Multi-Agent 솔루션

### 1.3 배경
- 기존 회의 AI 솔루션(Otter.ai, Fireflies, Clova Note)은 **회의 전 준비** 또는 **회의 후 요약**에 집중
- 회의가 **진행되는 중간**에 실시간으로 개입하여 품질을 높여주는 솔루션 부재
- 기업들은 회의 문화 개선을 위해 퍼실리테이션 교육/컨설팅에 비용 지출하지만 지속적 실행 어려움

### 1.4 Value Proposition
> "AI 퍼실리테이터가 모든 회의에 함께합니다"

| 기존 솔루션 | MeetingMod |
|------------|------------|
| Passive (녹음/요약) | Active (실시간 개입/코칭) |
| 회의 "기록" | 회의 "개선" |

---

## 2. 타겟 사용자

### 2.1 Primary Persona
- 스타트업/중소기업 팀 리더
- 회의 문화 개선을 원하는 조직
- Agile/수평적 문화를 지향하는 기업

### 2.2 User Pain Points
| 단계 | 문제점 |
|------|--------|
| **회의 전** | 참석자들이 안건을 읽지 않고 참석 → 맥락 공유에 시간 낭비 |
| **회의 중** | 주제 이탈(삼천포), 특정인 독점 발언, Top-down 의사결정 고착화 |
| **회의 후** | Action Item 누락, 회의 품질 피드백 부재, 개인별 개선점 미제공 |

## 3. 핵심 기능 요구사항

### 3.1 회의 전 (Pre-Meeting)

#### F1: 회의 준비 자료 입력
- **입력 방식**: Markdown 텍스트 에디터
- **필수 입력 항목**:
  - 참석자 목록 (이름, 역할)
  - 아젠다 (논의 주제)
  - 참고 자료 (링크, 문서)
- **회의 원칙 설정**: 별도 페이지에서 편집 가능
- **액션 버튼**:
  - `회의 시작` 버튼: 회의 모드 진입

#### F2: 회의 원칙 관리 페이지
- Agile 원칙, AWS Leadership Principles 등 사전 정의된 템플릿 제공
- 사용자가 직접 원칙 추가/수정/삭제 가능
- 회의별로 적용할 원칙 선택

### 3.2 회의 중 (During Meeting)

#### F3: 실시간 음성 인식
- **입력**: 맥북 마이크를 통한 실시간 오디오 캡처
- **처리**: OpenAI Whisper STT로 텍스트 변환
- **화자 분리**: 참석자 목록 기반 AI 자동 화자 분리
- **표시**: 실시간 자막 (화자명 + 발화 내용)

#### F4: 실시간 개입 (Core Feature)

**개입 타이밍**: 발화자가 말을 멈췄을 때 적극적으로 개입

**개입 출력 방식**:
- 경고음 재생 (시청각 주의 환기)
- 화면 Toast 메시지 표시

| 상황 | Agent 개입 |
|------|-----------|
| **회의 원칙 위반** | 설정된 원칙(Agile, AWS LP)에 벗어나는 발언 시 지적 |
| **주제 이탈** | Parking Lot 처리, 원래 주제로 복귀 유도 |
| **발언 불균형** | 발언 분포 확인 후 참여 독려 |
| **Top-down 감지** | 수평적 의사결정 원칙 적용 시 다른 의견 요청 |

#### F5: 발언 통계
- 참석자별 발언 시간/횟수 실시간 집계
- 발언 비율 시각화 (Progress Bar)
- 불균형 감지 시 Agent 개입 트리거

#### F6: 회의 종료 처리
- 회의 내용 리캡 생성
- Action Item 자동 추출 및 수행자 선정

### 3.3 회의 후 (Post-Meeting)

#### F7: 회의록 저장
- **저장 위치**: `meetings/{meeting-id}/` 디렉토리
- **저장 형식**: Markdown 파일
- **저장 항목**:
  - `preparation.md`: 회의 준비 자료
  - `transcript.md`: 전체 녹취록
  - `interventions.md`: Agent 개입 기록
  - `summary.md`: 회의 요약
  - `action-items.md`: Action Items

#### F8: 개인별 피드백
- 회의 원칙 준수도 기반 피드백
- 참여도 기반 피드백
- 개선점 제안

## 4. 기업 회의 문화 설정

### 4.1 회의 원칙 템플릿

**Agile 원칙**:
1. 수평적 의사결정: 모든 참석자의 의견을 동등하게 존중
2. 타임박스: 정해진 시간 내 논의 완료
3. Action-oriented: 모든 논의는 Action Item으로 연결
4. 짧고 집중: 불필요한 발언 최소화

**AWS Leadership Principles**:
1. Customer Obsession: 고객 관점에서 논의
2. Ownership: 책임감 있는 의견 제시
3. Disagree and Commit: 이견 표출 후 결정 따르기
4. Have Backbone; Disagree: 동의하지 않으면 정중히 반박
5. Dive Deep: 세부사항까지 파악
6. Bias for Action: 빠른 결정, 실행 우선

### 4.2 원칙 위반 감지 방식
- **권장**: LLM 직접 판단 (GPT-5.2)
- 발화 내용 + 설정된 원칙을 컨텍스트로 전달
- 위반 여부 및 구체적 피드백 생성

## 6. 해커톤 MVP 범위

### 6.1 Must Have (데모 핵심)
- [ ] 회의 준비 자료 입력 UI (Markdown Editor)
- [ ] 회의 원칙 설정/편집 페이지
- [ ] 실시간 음성 인식 (Whisper STT, 실제 마이크 입력)
- [ ] AI 자동 화자 분리 (참석자 목록 기반)
- [ ] Moderator Agent 실시간 개입
  - 경고음 + Toast 메시지
  - 발화 멈춤 시점에 적극 개입
  - 주제 이탈 감지 & 복귀 유도
  - 발언 분포 모니터링 & 참여 독려
  - 회의 원칙 위반 지적
- [ ] 회의 종료 시 리캡 & Action Item 추출
- [ ] 회의록 Markdown 파일 저장

### 6.2 Out of Scope (해커톤 범위 외)
- 이메일 발송 연동
- 준비도 체크 퀴즈
- 화상회의 툴 연동 (Zoom, Meet)

## 7. 기술 스택 결정사항

| 항목 | 결정 |
|------|------|
| **Backend** | Python 3.12 + OpenAI Agents SDK |
| **Web Server** | FastAPI (필요시) |
| **Frontend** | Next.js + React + shadcn/ui |
| **STT** | **OpenAI Realtime API** (실시간 음성 인식) |
| **LLM** | **GPT-5.2** (Agent 추론, 화자 분리, 원칙 위반 감지) |
| **Storage** | 회의별 디렉토리 + .md 파일 |
| **Alert Sound** | 짧고 부드러운 차임벨 (1초 이내) |

## 8. 성공 지표 (해커톤 평가 관점)

1. **기술적 완성도**: Multi-Agent 아키텍처가 실제로 동작하는가
2. **실시간성**: 회의 중 개입이 자연스럽고 시의적절한가
3. **차별성**: 기존 솔루션과 명확히 다른 가치를 제공하는가
4. **비즈니스 가능성**: 실제 기업에서 사용 가능한 솔루션인가
